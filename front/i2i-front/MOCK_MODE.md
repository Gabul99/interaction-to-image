# Mock 모드 사용 가이드

서버를 돌릴 수 없는 상황에서 프론트엔드를 테스트하기 위한 Mock 모드 사용법입니다.

## Mock 모드 활성화

### 방법 1: 환경 변수 사용 (권장)

프로젝트 루트에 `.env` 파일을 생성하거나 수정:

```bash
VITE_USE_MOCK_MODE=true
```

그리고 개발 서버를 실행:

```bash
npm run dev
```

### 방법 2: 코드에서 직접 수정

`src/config/api.ts` 파일을 열고:

```typescript
export const USE_MOCK_MODE = true; // false를 true로 변경
```

## Mock 모드 동작

Mock 모드가 활성화되면:

1. **실제 서버 연결 없음**: WebSocket 연결을 시도하지 않습니다
2. **시뮬레이션 실행**: 더미 데이터로 이미지 생성 과정을 시뮬레이션합니다
3. **모든 기능 동작**: 이미지 생성, 브랜치 생성 등이 모두 시뮬레이션으로 동작합니다

## Mock 모드에서 동작하는 기능

- ✅ 프롬프트 입력 및 객체 리스트 생성 (더미 데이터)
- ✅ 구도 설정
- ✅ 이미지 생성 시작 (더미 이미지 스트림)
- ✅ 브랜치 생성 (더미 이미지 스트림)
- ✅ 그래프 시각화
- ✅ 노드 선택 및 드래그

## Mock 모드 비활성화

실제 서버와 연결하려면:

1. `.env` 파일에서 `VITE_USE_MOCK_MODE=false` 또는 삭제
2. 또는 `src/config/api.ts`에서 `USE_MOCK_MODE = false`로 설정

## 주의사항

- Mock 모드에서는 실제 서버의 응답을 받지 않으므로, 백엔드 개발 시에는 Mock 모드를 비활성화해야 합니다
- Mock 모드와 실제 서버 모드를 자주 전환하면서 테스트할 수 있습니다
- 모든 실제 서버 연결 코드는 그대로 유지되며, Mock 모드일 때만 시뮬레이션이 실행됩니다
